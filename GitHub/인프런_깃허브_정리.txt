git : 버전관리
github : 클라우드 저장소

* 다수의 사람이 하나의 프로젝트에 대해 버전 관리를 해야 한다면?

git
- 원하는 시점마다 버전을 만들고 이들 간에 자유롭게 이동 가능
- 동료의 버전과 내 버전을 비교해 최신 상태로 업데이트 가능

git은 어디에서든 쓸 수 있다
- 개인 컴ㅌ
- USB
- 회사 서버
- 클라우드 (Github, BitBucket, GitLab..)

git은 CLI / GUI 에서 사용 가느함.

github에 코드를 올리는 과정
1. 특정 폴더에 '여기애서 git을 쓸거임'이라고 알려준다
2. 코딩
3. 변경한 파일 중 올리기를원하는 것만 선택
4. 선택한 파일들을 한 덩어리로 만들고 설명 적기
5. github 사이트에 프로젝트 저장소 만들기
6. 폴더에 github 저장소 주소 알려주기
7. 폴더 내 올리기 원했던 파일들을 github에 올리기


환경설정은 어떻게 할까?
- 원하는 폴더로 이동하여 git init한다
- git init하면 .git 디렉토리 생성됨
	-> 이게 로컬 저장소
- 로컬 저장소에 내가 만든 버전 정보, 원격 저장소 주소 등이 저장된다
- 원격 저장소에서 내 컴퓨터로 코드를 받아오면 로컬 저장소가 자동으로 생긴다
- 한 폴더에 하나의 로컬 저장소만 유지해야 한다
